Comment aplatir un JSON imbriqué — Transformer des données API complexes en CSV / Excel propre en quelques secondes

Publié le 28 avril 2026 · Mis à jour le 18 mai 2026 — 8 min de lecture

Presque chaque API renvoie du JSON imbriqué. Vous demandez une liste d'utilisateurs, et dans chaque utilisateur se trouve un objet adresse avec rue, ville, code postal ; peut-être un objet préférences, ou un tableau de commandes. Les tableurs et bases de données, eux, attendent des tables plates. C'est là que l'aplatissement du JSON imbriqué devient essentiel.

Que vous ayez besoin de convertir un tableau JSON en CSV, d'exporter du JSON vers Excel en ligne, ou de transformer une réponse d'API brute en un CSV formaté qu'Excel peut lire avec les accents corrects, ce guide vous montrera la méthode la plus rapide et la plus privée — sans écrire une seule ligne de code.

Que signifie réellement « aplatir un JSON imbriqué » ?

Cela signifie prendre une structure comme celle-ci : 

[
  {
    "nom": "Alice",
    "adresse": {
      "ville": "Paris",
      "code_postal": "75001"
    }
  }
]

Et la transformer en tableau plat : 

nom,adresse.ville,adresse.code_postal
Alice,Paris,75001

Le sous‑objet adresse disparaît ; ses champs deviennent de nouvelles colonnes avec des en‑têtes en notation par points. C'est la base du JSON vers Excel en ligne, et c'est exactement ce que fait notre Convertisseur JSON en CSV automatiquement lorsque vous cochez l'option Aplatir les objets.

Pourquoi aplatir du JSON ? Raisons concrètes

  • Convertir du JSON d'API en CSV pour des rapports métier. Votre backend envoie des données imbriquées ; votre interlocuteur veut un tableau.
  • Préparer des données pour des bases de données. La plupart des outils SQL importent mieux du CSV plat que du JSON profondément imbriqué.
  • Déboguer des jetons JWT. Souvent, la charge utile d'un JWT contient des claims imbriqués. Vous pouvez décoder le JWT, puis aplatir sa charge utile en CSV pour analyse.
  • Partager avec des collègues non techniques. Ils connaissent Excel, pas les chemins JSON.

Utilisations courantes de l'aplatissement JSON

  • Exporter des réponses d'API vers Excel
  • Importer du JSON dans des bases de données SQL
  • Analyser des données JSON dans des tableurs
  • Créer des rapports CSV à partir d'API
  • Préparer du JSON pour des outils de BI

Comment fonctionne l'aplatissement en coulisses

Aplatir un objet JSON est un processus récursif. En partant du niveau supérieur, on parcourt chaque clé. Si la valeur est un autre objet (et pas un tableau), on descend plus profondément et on préfixe la clé enfant avec la clé parente, en les séparant par un point. Si la valeur est d'un type simple (chaîne, nombre, booléen, null), on la stocke comme colonne.

Voici l'algorithme en Python — quelques lignes seulement : 

def aplatir(obj, prefixe='', sep='.'):
    items = {}
    for cle, val in obj.items():
        nouvelle_cle = f"{prefixe}{sep}{cle}" if prefixe else cle
        if isinstance(val, dict):
            items.update(aplatir(val, nouvelle_cle, sep))
        else:
            items[nouvelle_cle] = val
    return items

Et la même logique en JavaScript, prête à tourner dans votre navigateur : 

function aplatir(obj, prefixe = '', sep = '.') {
  return Object.keys(obj).reduce((acc, cle) => {
    const nouvelleCle = prefixe ? `${prefixe}${sep}${cle}` : cle;
    if (typeof obj[cle] === 'object' && !Array.isArray(obj[cle]) && obj[cle] !== null) {
      Object.assign(acc, aplatir(obj[cle], nouvelleCle, sep));
    } else {
      acc[nouvelleCle] = obj[cle];
    }
    return acc;
  }, {});
}

Quand vous appliquez cela à un tableau d'objets (comme une réponse d'API), on aplatit d'abord chaque élément, puis on combine toutes les clés pour former l'en‑tête du CSV. C'est exactement ce que le convertisseur Devstoolsbox fait en temps réel.

Cas limites qui surprennent les développeurs

L'aplatissement est simple jusqu'à ce que vous rencontriez l'un de ces scénarios réels :

Tableaux d'objets

Si un champ contient un tableau d'objets — disons commandes: [{id: 1}, {id: 2}] — la fonction d'aplatissement de base ne peut pas transformer cela en une seule colonne. Notre outil produit chaque élément du tableau comme une nouvelle ligne, en dupliquant les champs parents. C'est souvent ce que l'on veut pour un tableau plat, mais cela peut générer beaucoup de lignes. Si vous préférez conserver le tableau sous forme de chaîne JSON dans la cellule, vous pouvez ajuster les options d'aplatissement dans le convertisseur.

Valeurs nulles

Que se passe-t-il quand un champ est null ? Dans Devstoolsbox, null devient une cellule vide. C'est propre pour Excel ; d'autres outils pourraient supprimer complètement la colonne. Vous pouvez aussi choisir de représenter les nulls par la chaîne "null" ou par 0 via les paramètres de gestion des cellules vides.

Points dans les noms de clés d'origine

Si votre JSON contient déjà des clés comme "prenom.nom", l'aplatissement avec un séparateur point créera une collision. Notre outil vous avertit quand il détecte des points dans les clés. Vous pouvez passer le séparateur à __ ou renommer ces clés avant l'aplatissement.

Profondeur maximale

Bien qu'il n'y ait pas de limite stricte, une imbrication extrêmement profonde (plus de 10 niveaux) peut produire des centaines de colonnes. Dans ce cas, vous pouvez choisir d'aplatir seulement les premiers niveaux et de conserver le reste sous forme de chaîne JSON sérialisée. Le convertisseur vous permet de définir une profondeur d'aplatissement maximale.

Aplatir ou normaliser : de quoi avez-vous vraiment besoin ?

On confond souvent aplatissement et normalisation. Ils servent des objectifs différents.

L'aplatissement produit une seule table large. Chaque objet imbriqué devient des colonnes supplémentaires ; chaque élément de tableau devient une ligne supplémentaire avec les données parentes répétées. C'est idéal pour des exports rapides, des analyses Excel et des rapports.

La normalisation divise les données en plusieurs tables liées — par exemple, un utilisateurs.csv et un commandes.csv reliés par un utilisateur_id. C'est l'approche propre pour les bases de données relationnelles.

Quand vous aplatissez un utilisateur avec ses commandes, vous obtenez : 

nom,commande.id,commande.montant
Alice,1,50
Alice,2,30

Si vous normalisez, vous exportez deux CSVs : 

# utilisateurs.csv
id,nom
1,Alice

# commandes.csv
utilisateur_id,id,montant
1,1,50
1,2,30

Le choix dépend de l'outil cible. Les utilisateurs d'Excel adorent une table plate ; les administrateurs de bases de données préfèrent des tables normalisées.

Pour les gros fichiers : pandas et jq

L'outil en ligne brille pour un aplatissement rapide et interactif. Mais quand vous avez affaire à un export JSON de plusieurs gigaoctets, vous voudrez une approche en ligne de commande. Voici deux méthodes éprouvées.

Python avec pandas

Les analystes de données adorent pd.json_normalize. Cela fait exactement ce que notre outil fait, mais de manière scriptable. Ce bout de code lit un fichier JSON, aplatit le tableau à data.users, et écrit un CSV avec le BOM UTF‑8 pour Excel : 

import pandas as pd
import json

with open('data.json') as f:
    data = json.load(f)

df = pd.json_normalize(data['data']['users'], sep='.')
df.to_csv('output.csv', index=False, encoding='utf-8-sig')

utf-8-sig est la façon dont Python ajoute le BOM UTF‑8. Ce petit détail fait gagner des heures quand les collègues ouvrent le fichier dans Excel.

jq pour le terminal

jq est un processeur JSON léger et incroyablement rapide. Pour extraire et aplatir un sous‑ensemble de champs d'un grand tableau imbriqué, vous pouvez lancer : 

jq -r '.data.users[] | [.nom, .adresse.ville, .adresse.code_postal] | @csv' data.json

Cela envoie les lignes aplaties directement vers stdout, que vous pouvez rediriger vers un fichier CSV. Idéal pour les pipelines automatisés.

Foire Aux Questions

Pourquoi mon CSV aplati contient-il des lignes répétées pour le même utilisateur ?

Cela se produit quand vous aplatissez un objet JSON qui contient un tableau (comme des commandes). Chaque élément du tableau crée une nouvelle ligne où les colonnes parentes sont répétées. Si vous voulez éviter cette duplication, vous pouvez soit exporter plusieurs fichiers CSV (normalisation), soit choisir comment les tableaux sont représentés (chaîne JSON, premier élément uniquement…).

Quelle est la différence entre aplatir et normaliser du JSON ?

L'aplatissement crée une seule table large ; la normalisation divise les données en plusieurs tables liées. Utilisez l'aplatissement pour un export rapide, la normalisation pour un import en base de données.

Puis-je aplatir un JSON qui contient des points dans les noms de clés ?

C'est risqué car les points servent de séparateurs. Notre outil détecte les points dans les clés d'origine et vous avertit. Vous pouvez choisir un autre séparateur (comme __) ou renommer les clés avant l'aplatissement.

Que deviennent les valeurs null quand j'aplatis du JSON ?

Dans notre outil, les valeurs null deviennent des cellules vides. C'est généralement ce que l'on souhaite. Si vous avez besoin de chaînes 'null' explicites ou de zéros, ajustez la gestion des cellules vides dans les paramètres du convertisseur.

Comment aplatir des fichiers JSON très volumineux (plusieurs gigaoctets) ?

Utilisez des outils en ligne de commande comme jq ou Python avec pandas. Nous montrons des exemples dans l'article. Notre outil en ligne fonctionne idéalement jusqu'à quelques mégaoctets de JSON, entièrement dans votre navigateur.
