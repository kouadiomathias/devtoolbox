Comparer Deux Textes en Ligne

Publié le 3 mai 2026 · Mis à jour le 18 mai 2026 — Lecture 9 min
Notre comparateur de texte (Text Diff) affiche instantanément les différences entre deux contenus : lignes ajoutées en vert, supprimées en rouge. Il fonctionne entièrement dans votre navigateur, sans inscription, et permet même de partager un lien unique de votre diff. Aucune donnée n'est transmise à un serveur.

La commande indique que le build est propre. Le serveur, lui, dit le contraire. Quelque part entre la configuration que vous croyez avoir déployée et celle qui s'exécute réellement, une ligne a changé — et la repérer à l'œil nu prend plus de temps qu'il ne le devrait.

Comment comparer deux textes pas à pas

  1. Accédez à l'outil : rendez-vous sur la page Comparateur de texte.
  2. Collez vos deux versions : dans la zone de gauche, placez le texte original ; à droite, la version modifiée. Vous pouvez aussi glisser-déposer un fichier ou utiliser les boutons « Coller ».
  3. Réglez les options (facultatif) : choisissez d'ignorer la casse, les espaces en fin de ligne ou les lignes vides. Sélectionnez un langage pour la coloration syntaxique (JSON, SQL, Python…), ou laissez l'auto‑détection.
  4. Lancez la comparaison : cliquez sur Comparer. Le résultat s'affiche immédiatement dans le panneau inférieur.
  5. Lisez le résultat : les lignes ajoutées apparaissent en vert, les supprimées en rouge. Dans la vue fractionnée, les deux textes sont côte à côte ; dans la vue unifiée (comme git diff), un seul flux avec des marqueurs + et -.
  6. Partagez ou copiez : utilisez le bouton Partager URL pour générer un lien unique que vous pouvez envoyer à un collègue. Il retrouvera exactement le même diff.

Temps d'exécution : inférieur à 100 ms pour des textes de quelques centaines de lignes.
Taille maximale recommandée : 5 Mo (au‑delà, utilisez un outil en ligne de commande).
Formats de sortie : affichage HTML interactif (avec surlignage), possibilité de copier le texte brut ou de partager l'URL.

Astuces de pro

  • Avant de comparer, formatez : si vous comparez du JSON, utilisez d'abord notre formateur JSON pour indenter les deux versions de manière cohérente. Le diff sera bien plus lisible.
  • Utilisez la vue unifiée pour les patchs : si vous devez appliquer les modifications à un fichier de code, la vue unifiée (onglet « unifié ») vous donne un format proche de celui de git apply.
  • Partagez sans risque : le lien de partage encode tout dans l'URL. Aucune donnée n'est stockée sur un serveur. Vous pouvez envoyer le lien même sur un réseau non sécurisé.

Diff ligne par ligne ou mot à mot : lequel choisir ?

La plupart des outils, y compris la commande diff et Git, comparent les textes ligne par ligne. Mais quand une seule ligne contient une petite modification — par exemple une adresse IP corrigée ou une faute de frappe dans un long paragraphe — un diff mot à mot devient bien plus utile. Il surligne uniquement les caractères ou mots qui ont changé à l'intérieur de la ligne, plutôt que de marquer toute la ligne comme modifiée.

Notre comparateur propose les deux modes. Passez en diff mot à mot quand vous relisez des fichiers de configuration, de la documentation, ou tout contenu où une même ligne peut contenir plusieurs modifications indépendantes. C'est l'équivalent du git diff --word-diff, directement dans votre navigateur.

Que se passe-t-il en coulisses ? Un aperçu des algorithmes de diff

Quand vous comparez deux textes, l'outil ne se contente pas de les aligner mot à mot. Il résout un problème classique d'informatique : trouver la plus longue sous-séquence commune (LCS). La LCS est la plus longue séquence de lignes (ou de mots) qui apparaît dans les deux textes dans le même ordre, en autorisant des sauts. Une fois cette séquence commune identifiée, tout ce qui se trouve en dehors est marqué comme ajout ou suppression.

Différents outils peuvent produire des diffs légèrement différents car la LCS n'est pas toujours unique. Par exemple, si vous supprimez une ligne et en ajoutez une similaire ailleurs, l'algorithme peut interpréter le changement comme une modification plutôt qu'une suppression suivie d'une insertion — selon la façon dont il a été paramétré. C'est pourquoi git diff, VS Code et un outil en ligne peuvent afficher la même modification de manière différente. Notre comparateur utilise une variante qui a tendance à mieux aligner les blocs de code, mais pour les patchs de versionnage, la vue unifiée respecte le comportement attendu.

Au‑delà du texte : quand un simple diff ne suffit pas

Les diffs textuels fonctionnent parfaitement pour le code source, les fichiers de configuration et la prose. Mais ils atteignent leurs limites avec les données structurées ou binaires. Voici comment réagir quand une comparaison ligne à ligne ne convient plus :

  • Diff sémantique JSON : les objets JSON sont des dictionnaires non ordonnés. Deux fichiers JSON identiques avec un ordre de clés différent apparaîtront comme complètement modifiés dans un diff textuel. Utilisez un comparateur JSON sémantique comme jq associé à diff sur une version triée, ou notre propre formateur JSON après avoir ordonné les clés.
  • Diff d'images : des outils comme imagemagick (commande compare) ou des comparateurs d'images en ligne superposent les pixels différents. Exemple en ligne de commande : magick compare image1.png image2.png diff.png.
  • Diff de PDF : les PDF ne sont pas des fichiers texte. Utilisez diffpdf (interface graphique) ou pdftotext suivi de diff pour extraire le texte, ou encore la fonction de comparaison d'Adobe Acrobat.
  • Diff de bases de données : pour comparer des schémas ou des données, tournez-vous vers mysqldiff (MySQL), pgdiff (PostgreSQL) ou des outils graphiques comme DBeaver.

Pour le quotidien du développement, un diff textuel couvre 90 % des besoins. Mais savoir quand passer à un comparateur plus approprié fait gagner du temps et évite bien des confusions.

La commande diff en ligne de commande : la boîte à outils du power user

Quand vous êtes déjà dans un terminal, ou que les fichiers sont trop volumineux pour le navigateur, la commande diff est votre meilleure alliée. Voici les options que j'utilise tous les jours :

# Comparaison simple
diff fichier1.txt fichier2.txt

# Ignorer la casse
diff -i fichier1.txt fichier2.txt

# Ignorer les changements d'espaces
diff -w fichier1.txt fichier2.txt

# Sortie unifiée (comme git diff)
diff -u fichier1.txt fichier2.txt

# Comparer récursivement deux dossiers
diff -r dossier1/ dossier2/

L'option -u produit un diff unifié, exactement celui qu'affiche Git. Il est plus lisible et peut servir de patch. Si vous comparez des dossiers entiers, diff -r vous indique rapidement quels fichiers ont changé.

Git diff avancé : au‑delà des basiques

Les utilisateurs de Git emploient souvent git diff sans en exploiter toutes les possibilités. Voici quelques commandes qui vont plus loin que la vue ligne à ligne par défaut :

# Afficher les changements mot à mot
git diff --word-diff

# Ignorer uniquement les espaces
git diff --ignore-all-space

# Comparer avec le commit précédent
git diff HEAD~1

# Comparer deux branches
git diff main..feature-branch

# Voir ce qui a changé pour un fichier spécifique entre deux branches
git diff main..feature-branch -- chemin/vers/fichier

Le diff mot à mot est particulièrement précieux pour les revues de code : il identifie exactement la variable ou le paramètre modifié, sans encombrer le diff avec les parties inchangées de la ligne. Et comparer deux branches vous permet d'examiner toute une branche de fonctionnalité d'un seul coup — vous pouvez ensuite utiliser notre outil en ligne pour isoler un fichier précis.

Cas d'usage concrets

Notre comparateur est utilisé quotidiennement par des développeurs pour relire une pull request, par des administrateurs système pour vérifier deux fichiers de configuration .env, ou par des rédacteurs techniques pour suivre l'évolution d'un article. Je l'utilise personnellement pour vérifier que ma dernière mise à jour d'un script d'automatisation n'a pas modifié de ligne critique, sans avoir à ouvrir un terminal.

Autre exemple : un collègue data analyst compare deux exports CSV de sa base client avant et après migration. Il repère immédiatement les colonnes ajoutées ou les valeurs modifiées sans avoir à lancer Excel.

Confiance et sécurité des données

Le comparateur de texte fonctionne intégralement dans votre navigateur. Aucune ligne de texte n'est transmise à un serveur distant. Même la fonction de partage d'URL encode toutes les informations dans l'URL elle‑même, sans jamais les stocker. Vous pouvez utiliser l'outil pour des documents confidentiels en toute tranquillité, car rien ne quitte votre machine.
