Comment créer un effet Glassmorphism en CSS – Exemples concrets et astuces de pro

Publié le 28 avril 2026 · Mis à jour le 18 mai 2026 — 10 min de lecture

Le glassmorphism s'est invité partout : tableaux de bord, pages d'atterrissage, applications mobiles. Ces panneaux en verre dépoli qui floutent l'arrière-plan juste ce qu'il faut pour donner de la profondeur se construisent avec du CSS pur, sans plugin. Dans ce guide, nous allons monter une carte glass étape par étape, puis creuser les détails qui piègent la plupart des développeurs : la différence entre backdrop-filter et filter, les performances sur mobile, les pièges du contexte d'empilement, et comment garder l'effet accessible. Vous trouverez des extraits de code prêts à l'emploi et un lien vers notre générateur visuel gratuit qui fait tout le travail à votre place.

Les trois ingrédients du glassmorphism

Un effet verre convaincant repose toujours sur :

  • Un fond semi‑transparent, généralement blanc ou d'une teinte très claire.
  • Un flou backdrop-filter qui gèle le contenu derrière l'élément.
  • Une bordure subtile qui définit le bord du verre.

La carte glass CSS de base

Voici la version la plus courte qui fonctionne sur tous les navigateurs modernes : 

.carte-verre {
  background: rgba(255, 255, 255, 0.2);
  backdrop-filter: blur(12px);
  -webkit-backdrop-filter: blur(12px); /* Safari */
  border-radius: 20px;
  border: 1px solid rgba(255, 255, 255, 0.3);
  padding: 28px;
  color: #fff;
}

Décortiquons chaque propriété. background: rgba(255,255,255,0.2) donne cette couche blanche presque invisible. La valeur alpha contrôle la transparence. backdrop-filter: blur(12px) applique un flou gaussien à tout ce qui se trouve derrière l'élément. Le préfixe -webkit- reste nécessaire pour Safari. Enfin, la bordure ajoute un léger liseré vitreux ; sans elle, la carte peut se fondre dans le fond.

backdrop-filter vs filter : une distinction cruciale

Beaucoup de débutants confondent backdrop-filter et filter. Ils sont fondamentalement différents, et utiliser le mauvais casse complètement l'effet verre.

  • filter s'applique à l'élément lui-même et à tous ses enfants. Si vous écrivez filter: blur(5px), la carte et le texte à l'intérieur deviendront flous — ce n'est pas ce que vous voulez.
  • backdrop-filter n'affecte que ce qui se trouve derrière l'élément. La carte elle-même reste nette ; seul l'arrière-plan reçoit l'effet givré.

Une comparaison côte à côte rend la chose évidente :

/* ❌ mauvais pour le verre : toute la carte devient floue */
.carte-floue {
  filter: blur(8px);
}

/* ✅ correct : seul l'arrière-plan est flouté */
.carte-verre {
  backdrop-filter: blur(12px);
}

Retenez donc : backdrop-filter pour le verre, filter pour manipuler l'élément lui-même (comme le niveau de gris ou l'opacité).

Performance : quand le flou fait chuter le framerate

backdrop-filter force le navigateur à créer un nouveau calque composite. Ce calque est redessiné chaque fois que l'arrière-plan change, ce qui peut être coûteux sur mobile ou lors d'animations. Un flou important (20px+) sur une grande carte peut faire chuter les images, surtout sur les téléphones d'entrée de gamme.

Règles pratiques pour des effets verre fluides :

  • Limitez le flou à 12-15px pour les éléments animés ou qui défilent. Les cartes statiques peuvent aller plus haut.
  • Utilisez will-change: backdrop-filter avec parcimonie — cela indique au navigateur de préparer un calque séparé, mais en abuser gaspille la mémoire GPU.
  • Testez toujours sur un appareil réel, pas seulement dans Chrome DevTools. Ouvrez l'onglet Performance et surveillez les repaints des calques composites.

Le piège du contexte d'empilement (et pourquoi z-index: -1 casse tout)

Un bug subtil mais courant : backdrop-filter crée un contexte d'empilement. Cela signifie que tout élément derrière la carte glass avec un z-index négatif risque d'être masqué, même si vous vous attendiez à ce qu'il reste visible. Cela arrive parce que l'élément glass et l'arrière-plan en z-index négatif appartiennent désormais à des couches d'empilement différentes.

Par exemple, si vous placez un fond en dégradé avec z-index: -1 puis une carte glass par-dessus, l'arrière-plan peut disparaître derrière la carte — non pas à cause du flou, mais à cause du contexte d'empilement. La solution ? Évitez les z-index négatifs sur les arrière-plans ; placez plutôt le fond dans un conteneur avec un z-index positif ou fiez-vous à l'ordre naturel du DOM.

Glassmorphism vs neumorphism vs skeuomorphisme

Le glassmorphism est souvent cité aux côtés du neumorphism et du skeuomorphisme. Tous trois créent de la profondeur, mais avec des philosophies différentes.

Utilisez le glassmorphism lorsque vous voulez de la profondeur tout en montrant le contenu d'arrière-plan. Le neumorphism convient aux contrôles simples et statiques. Le skeuomorphisme est surtout historique aujourd'hui.

Carte glass en Tailwind CSS

Si vous travaillez avec Tailwind, vous pouvez obtenir le même effet avec des classes utilitaires. Voici une carte glass en Tailwind pur : 

<div class="backdrop-blur-md bg-white/20 border border-white/30 rounded-2xl p-7">
  <h2 class="text-white text-xl font-semibold">Carte Glass</h2>
  <p class="text-white/80">Construite avec Tailwind.</p>
</div>

backdrop-blur-md correspond à un flou moyen (12px par défaut). Vous pouvez utiliser backdrop-blur-sm ou backdrop-blur-lg selon vos besoins.

Composant React avec styled‑components

Pour créer une carte glass réutilisable dans un projet React, vous pouvez définir un composant stylisé avec une opacité et un flou ajustables : 

import styled from 'styled-components';

const CarteVerre = styled.div`
  background: rgba(255, 255, 255, ${props => props.opacite || 0.2});
  backdrop-filter: blur(${props => props.flou || 12}px);
  -webkit-backdrop-filter: blur(${props => props.flou || 12}px);
  border-radius: 20px;
  border: 1px solid rgba(255, 255, 255, 0.3);
  padding: 28px;
  color: white;
`;

// Utilisation
<CarteVerre opacite={0.25} flou={16}>
  <h2>Tableau de bord</h2>
</CarteVerre>

Fallback progressif pour les navigateurs sans support

Plutôt que de laisser les utilisateurs d'IE11 avec une mise en page cassée, fournissez un fond solide semi‑opaque comme solution de repli. La règle @supports permet d'écrire une amélioration progressive propre : 

.carte-verre {
  /* Fallback : solide mais translucide */
  background: rgba(255, 255, 255, 0.7);
}

@supports (backdrop-filter: blur(1px)) {
  .carte-verre {
    background: rgba(255, 255, 255, 0.2);
    backdrop-filter: blur(12px);
    -webkit-backdrop-filter: blur(12px);
  }
}

Ainsi, chaque navigateur reçoit un design utilisable. IE11 voit une carte blanche solide ; les navigateurs modernes profitent du verre dépoli.

Accessibilité : rendre le verre lisible pour tous

backdrop-filter crée un problème de contraste dynamique. Le texte repose sur un fond dont la luminosité dépend de ce qui se trouve derrière le verre — une image sombre peut offrir un contraste suffisant, tandis qu'un dégradé clair peut ne pas le faire. Les vérificateurs de contraste statiques ne peuvent pas capturer cela.

Voici comment garder vos panneaux glass accessibles :

  • Testez la carte avec le fond le plus clair possible qu'elle pourra rencontrer. Si le texte passe le contraste WCAG AA dans ce pire scénario, c'est bon.
  • Augmentez légèrement l'opacité du fond (0.3–0.4) si la lisibilité est faible.
  • Ajoutez une ombre de texte subtile pour améliorer la lisibilité, par ex. text-shadow: 0 1px 3px rgba(0,0,0,0.5);

Foire Aux Questions

Pourquoi mon effet glassmorphism se casse-t-il quand j'utilise z-index: -1 ?

backdrop-filter crée un nouveau contexte d'empilement. Les éléments avec un z-index négatif derrière la carte glass peuvent être masqués. Plutôt que d'utiliser un z-index négatif, restructurez le HTML pour que l'arrière-plan soit naturellement derrière la carte.

Est-ce que backdrop-filter est lourd sur mobile ?

Il peut l'être. Sur les appareils peu puissants, un flou important peut faire chuter les images. Limitez le flou à 12px pour les éléments animés, utilisez will-change uniquement quand c'est nécessaire, et testez sur du matériel réel.

Comment rendre le glassmorphism accessible ?

Comme le contraste est dynamique, testez avec le fond le plus clair possible. Si besoin, augmentez l'opacité du fond ou ajoutez une ombre de texte subtile. Prévisualisez toujours sur de vrais fonds.
