Générateur de formes CSS : créer des clip-path, blobs et border-radius sans coder

Tu passes 20 minutes à calculer des coordonnées clip-path à la main pour un simple hexagone — et le résultat ne ressemble à rien. Dans ce guide, je te montre comment générer n'importe quelle forme visuellement, comprendre le code produit, et l'intégrer en CSS pur, Tailwind ou React en moins d'une minute.

Pourquoi des formes CSS plutôt que des images ?

La question revient souvent : pourquoi s'embêter avec clip-path quand on peut exporter une forme depuis Figma en PNG ? Trois raisons concrètes :

  • Zéro requête HTTP supplémentaire. Un clip-path pèse quelques octets de CSS. Une image PNG de la même forme peut facilement peser 8 à 15 Ko — plus le temps de téléchargement, plus la gestion des résolutions écran.
  • Responsive sans effort. Les valeurs en pourcentage s'adaptent à la taille du conteneur. Pas de @2x, pas de srcset, pas de recadrage à gérer.
  • Animable nativement. On peut interpoler un clip-path avec transition ou @keyframes — à condition que les deux états aient le même nombre de points.

La vraie difficulté, c'est l'écriture manuelle. Une forme à 6 côtés demande 6 paires de coordonnées, et la moindre erreur décale tout. C'est là qu'un générateur visuel change vraiment la donne.

Mode Clip-Path : tutoriel pas à pas

Le mode Clip-Path découpe un élément selon un polygone personnalisé. Voici le workflow en 4 étapes.

1
Choisir le nombre de côtés
Le générateur démarre avec un pentagone (5 côtés). Ajuste le curseur de 3 à 12 selon la forme voulue. Triangle, hexagone, étoile — chaque valeur produit instantanément un aperçu.
2
Déformer en glissant les poignées
Clique et glisse chaque sommet directement sur l'aperçu. Le code se met à jour en temps réel — c'est la fonctionnalité clé : tu modèles la forme à la main sans toucher à un seul chiffre.
3
Ajouter de l'arrondi
Le curseur d'arrondi applique un inset() pour adoucir les angles. Parfait pour des formes entre polygone strict et blob organique.
4
Copier le code généré
Choisis le format (CSS, Tailwind ou React) puis clique sur Copier. Le code est directement utilisable dans ton projet.

Voici un exemple de code réel généré pour un hexagone irrégulier :

CSS pur — hexagone irrégulier 
.hero-shape {
  clip-path: polygon(
    50%  0%,
    93%  25%,
    93%  75%,
    50%  100%,
    7%   75%,
    7%   25%
  );
}
⚠ Piège fréquent : clip-path masque aussi le box-shadow et l'overflow. Pour une ombre portée qui suit le contour de la forme, utilise filter: drop-shadow() sur un div parent — contrairement à box-shadow, il respecte le tracé réel du clip-path.

Mode Blob : comprendre le bruit de Perlin

Les blobs sont ces formes molles et organiques qu'on retrouve partout dans les landing pages modernes. Le générateur ne se contente pas d'arrondir un cercle : il utilise un algorithme de bruit Simplex (Perlin noise) pour produire des contours naturels et non répétitifs — le même principe que les textures de terrain dans les jeux vidéo.

Les trois paramètres à connaître

  • Seed : le point de départ du générateur aléatoire. Changer le seed produit un blob complètement différent, avec les mêmes réglages de complexité.
  • Complexité (octaves) : valeur basse = forme simple et ronde ; valeur haute = contour très irrégulier avec petites aspérités.
  • Échelle : l'amplitude des déformations. Un blob très gonflé vers l'extérieur ou resserré vers le centre.

💡 Astuce : Pour utiliser un blob comme masque d'image, définis le path dans un <clipPath id="mon-blob"> SVG et applique clip-path: url(#mon-blob) à ton image. Le générateur propose cette option directement dans le panneau d'export.

Mode Border-Radius : la notation à 8 valeurs

border-radius accepte jusqu'à 8 valeurs séparées par un slash. Peu de développeurs connaissent cette syntaxe — pourtant, elle permet de créer des gouttes, des feuilles, des amibes, sans toucher à SVG.

Syntaxe complète 
/* TL = top-left, TR = top-right, BR = bottom-right, BL = bottom-left */
border-radius: TL TR BR BL / TL-vertical TR-vertical BR-vertical BL-vertical;

/* Exemple — forme feuille */
border-radius: 60% 40% 30% 70% / 60% 30% 70% 40%;

Les 4 valeurs avant le slash contrôlent le rayon horizontal de chaque coin. Les 4 après le slash contrôlent le rayon vertical. En jouant sur l'asymétrie entre les deux axes, on obtient des courbes qui n'ont plus rien d'un rectangle arrondi classique.

Export : quel format choisir selon ton stack ?

Une fois la forme créée, le panneau d'export propose trois formats. Le choix est simple :

Format Quand l'utiliser Syntaxe
CSS pur Tout projet sans framework utilitaire clip-path: polygon(…)
Tailwind CSS Tailwind v3+ avec bracket syntax [clip-path:polygon(…)]
React JSX Composant React avec style inline style={{ clipPath: '…' }}
React JSX — composant prêt à l'emploi 
const shapeStyle = {
  clipPath: 'polygon(50% 0%, 93% 25%, 93% 75%, 50% 100%, 7% 75%, 7% 25%)',
  width: '300px',
  height: '300px',
  background: 'linear-gradient(135deg, #7c3aed, #a855f7)',
};

export function HeroShape() {
  return <div style={shapeStyle} aria-hidden="true" />;
}

Simulateur daltonisme : un réflexe accessibilité

Le générateur intègre un simulateur de déficience visuelle basé sur le modèle de Brettel-Viénot. En un clic, tu appliques un filtre protanopie, deutéranopie ou tritanopie sur l'aperçu.

En pratique : lors d'un projet, j'ai testé une forme orange vif sur fond rouge — rendu parfait visuellement. En activant le filtre protanopie, la forme devenait quasiment invisible : les deux couleurs se fondaient en un même marron terne. Sans ce simulateur, j'aurais livré une interface inaccessible à 8 % des hommes.

💡 Règle d'accessibilité : ne jamais reposer uniquement sur la couleur pour véhiculer une information. Une forme, une bordure, un texte alternatif — il faut toujours un second signal visuel. Les WCAG 2.1 l'exigent au niveau AA.

Partage et presets : gagner encore plus de temps

Les presets (triangle, hexagone, étoile, bulle, vague, goutte…) servent de point de départ. Sélectionne le plus proche de ce que tu veux, puis affine avec les curseurs et les poignées.

Le bouton Share URL encode toute la configuration dans le lien — seed, mode, nombre de côtés, rotation. Envoie-le à un collègue ou retrouve-le dans tes favoris : il verra exactement la même forme, sans rien reconfigurer. Idéal pour les revues de design à distance.
