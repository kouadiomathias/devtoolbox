Convertisseur JSON / CSV : tout ce qu'il faut savoir

Ce convertisseur JSON CSV permet de transformer rapidement des données JSON en CSV pour Excel ou Google Sheets, et inversement. Il supporte les structures imbriquées, l'aplatissement automatique et différents délimiteurs.

Il permet également de passer du format JSON (utilisé par les APIs) au format CSV (compatible Excel, Google Sheets) et inversement. Grâce à l'aplatissement automatique des objets imbriqués, vous pouvez traiter des données complexes en un clic.

Foire Aux Questions

Quelle est la différence entre JSON et CSV ?

Le JSON (JavaScript Object Notation) est un format structuré utilisé par les APIs, capable de représenter des objets imbriqués, des tableaux et des types variés. Le CSV (Comma-Separated Values) est un format tabulaire simple, compatible Excel et Google Sheets. Le JSON est plus flexible pour les données complexes ; le CSV est plus efficace pour l'analyse et l'import de données.

Comment convertir un JSON en CSV facilement ?

Collez votre JSON dans la zone de saisie ou importez un fichier, puis cliquez sur "Convertir en CSV". L'outil transforme automatiquement vos données en tableau compatible avec Excel et Google Sheets.

Comment convertir un fichier CSV en JSON ?

Collez votre CSV ou importez-le, choisissez le délimiteur si nécessaire, puis cliquez sur "Convertir en JSON". Le résultat est généré instantanément sous forme de tableau JSON.

Comment gérer les tableaux imbriqués dans un JSON ?

Utilisez le champ "Chemin du tableau" pour pointer vers l'emplacement exact de votre tableau. Par exemple, si votre JSON a la structure {"data": {"users": [...]}}, saisissez data.users. L'outil extraira automatiquement le tableau cible.

Comment gérer les objets JSON imbriqués lors de la conversion ?

Activez l'option d'aplatissement (flatten). L'outil transforme les objets imbriqués en colonnes simples avec des clés séparées par des points (ex : adresse.ville), ce qui assure une compatibilité parfaite avec les fichiers CSV et Excel.

Pourquoi l'export CSV inclut-il un BOM UTF-8 ?

Le BOM (Byte Order Mark) indique à Excel que le fichier est encodé en UTF-8. Sans ce marqueur, Excel peut mal afficher les accents et caractères spéciaux. L'option "Télécharger CSV" ajoute automatiquement ce BOM pour une compatibilité parfaite avec Excel français et international.

Pourquoi mon CSV ne s'affiche pas correctement dans Excel ?

Excel peut mal interpréter l'encodage ou le délimiteur selon votre région. Utilisez l'option d'export avec BOM UTF-8 et choisissez le délimiteur adapté : virgule pour les systèmes anglais, point-virgule pour les systèmes français.

Quels délimiteurs CSV sont pris en charge ?

L'outil prend en charge les délimiteurs les plus courants : virgule (,), point-virgule (;), tabulation et pipe (|). Une détection automatique analyse votre contenu pour deviner le délimiteur le plus probable.

Comment sont traitées les valeurs nulles ou vides lors de la conversion ?

Vous pouvez choisir comment représenter les valeurs null, undefined ou chaînes vides lors de la conversion CSV vers JSON : chaîne vide, texte "null" ou valeur zéro. Ce réglage est accessible dans le menu dédié.

Puis-je utiliser cet outil pour convertir JSON vers Excel ?

Oui. En convertissant votre JSON en CSV avec l'option BOM UTF-8 activée, vous pouvez ouvrir le fichier directement dans Excel ou l'importer dans Google Sheets sans problème d'encodage.

Quelle est la taille maximale des fichiers supportés ?

Il n'y a pas de limite fixe : tout dépend de votre navigateur et de votre machine. En pratique, les fichiers jusqu'à plusieurs mégaoctets sont traités sans problème. Pour des fichiers très volumineux (plus de 50 Mo), un traitement local en ligne de commande sera plus adapté.

Mes données sont-elles envoyées à un serveur lors de la conversion ?

Non. Toutes les conversions sont effectuées directement dans votre navigateur via JavaScript. Aucune donnée ne quitte votre ordinateur, ce qui garantit une confidentialité totale même pour des données sensibles.
