package main

import (
    "fmt"
    "time"
)

func main() {
    // Timestamp Unix actuel en secondes
    maintenantSecondes := time.Now().Unix()

    // Timestamp Unix actuel en nanosecondes
    maintenantNano := time.Now().UnixNano()

    // Timestamp Unix actuel en millisecondes
    maintenantMs := time.Now().UnixMilli() // Go 1.17+

    // Secondes Unix → time.Time (UTC)
    t := time.Unix(maintenantSecondes, 0).UTC()

    // Format ISO 8601 (RFC3339)
    fmt.Println(t.Format(time.RFC3339))       // "2025-06-08T10:00:00Z"
    fmt.Println(t.Format(time.RFC3339Nano))   // Avec nanosecondes

    // Format RFC 2822 (standard email)
    fmt.Println(t.Format(time.RFC1123Z))

    // Avec fuseau horaire
    paris, _ := time.LoadLocation("Europe/Paris")
    tParis := t.In(paris)
    fmt.Println(tParis.Format("2006-01-02 15:04:05 MST"))

    // time.Time → timestamp Unix
    d := time.Date(2025, 6, 8, 10, 0, 0, 0, time.UTC)
    fmt.Println(d.Unix())  // 1749376800

    _ = maintenantNano
    _ = maintenantMs
}

Qu'est-ce qu'un timestamp Unix ?

Un timestamp Unix (également appelé temps Unix ou temps POSIX) est le nombre de secondes écoulées depuis le 1er janvier 1970 à 00:00:00 UTC — un point de référence appelé l'epoch Unix. Il est indépendant des fuseaux horaires, ce qui en fait le standard universel pour stocker et comparer des dates dans les bases de données, les API, les logs et les systèmes distribués.

Deux formats coexistent en pratique : les secondes (un nombre à 10 chiffres comme 1749376800) et les millisecondes (un nombre à 13 chiffres comme 1749376800000). Date.now() en JavaScript et System.currentTimeMillis() en Java renvoient des millisecondes. time.time() en Python et la plupart des appels système Unix renvoient des secondes. Cet outil détecte automatiquement les deux.

Timestamps UUID expliqués

UUID v1 — timestamp + adresse MAC

L'UUID v1 encode un timestamp de 60 bits comptant les intervalles de 100 nanosecondes depuis le 15 octobre 1582 (début du calendrier grégorien). Le timestamp est stocké dans trois champs non contigus : time_low (32 bits), time_mid (16 bits) et time_hi (12 bits + quartet de version). Leur réassemblage nécessite d'inverser cette disposition. Le décodeur UUID v1 ci-dessus le fait automatiquement.

UUID v7 — timestamp Unix dans les 48 premiers bits

L'UUID v7 (RFC 9562, mai 2024) utilise une approche beaucoup plus simple : les 48 premiers bits encodent directement le timestamp Unix en millisecondes. Cela rend les UUID naturellement triables par date de création, et l'extraction est triviale — il suffit de lire les 12 premiers caractères hexadécimaux et de convertir depuis l'hexadécimal.

Le problème de l'an 2038

Sur les systèmes 32 bits, les timestamps Unix sont stockés dans des entiers signés de 32 bits avec un maximum de 2 147 483 647, correspondant au 19 janvier 2038 à 03:14:07 UTC. Après cette date, la valeur déborde vers un grand nombre négatif. La plupart des systèmes modernes utilisent des entiers 64 bits (valides jusqu'en l'an 292 277 026 596), mais les systèmes embarqués et le code legacy peuvent encore être affectés.

Conversion Windows FILETIME

Le FILETIME Windows est un entier 64 bits comptant les intervalles de 100 nanosecondes depuis le 1er janvier 1601. Le décalage entre l'epoch Windows et l'epoch Unix est exactement de 11 644 473 600 secondes (ou 116 444 736 000 000 000 en unités de 100 ns).

Conversion : FILETIME = (unixSeconds + 11644473600) × 10 000 000

En .NET : DateTime.FromFileTimeUtc(filetime) ou DateTimeOffset.FromUnixTimeSeconds(unixSeconds)

Foire aux questions

Qu'est-ce qu'un timestamp Unix ?

Un timestamp Unix est le nombre de secondes écoulées depuis le 1er janvier 1970 à 00:00:00 UTC. Il est indépendant des fuseaux horaires et utilisé universellement dans les bases de données, les API et les logs pour stocker et comparer des dates sans ambiguïté.

Comment savoir si un timestamp est en secondes ou en millisecondes ?

Un nombre à 10 chiffres est presque certainement en secondes (valide jusqu'en 2286). Un nombre à 13 chiffres est en millisecondes. Cet outil détecte automatiquement : les valeurs supérieures à 10 000 000 000 sont traitées comme des millisecondes. En cas de doute, vérifiez le nombre de chiffres.

Puis-je extraire une date d'un UUID v1 ?

Oui. L'UUID v1 encode un timestamp de 60 bits comptant les intervalles de 100 nanosecondes depuis le 15 octobre 1582. La section "Décodeurs d'identifiants spéciaux" ci-dessus extrait ce timestamp et le convertit dans tous les formats de date. Il détecte également si l'adresse MAC contenue dans l'UUID est réelle ou générée aléatoirement.

Quelle est la différence entre UTC et GMT ?

UTC (Temps Universel Coordonné) est le standard international moderne maintenu par des horloges atomiques. GMT (Greenwich Mean Time) est un fuseau horaire. Pour la plupart des usages pratiques, ils sont équivalents — UTC+0 et GMT représentent la même heure. Cependant, UTC est le standard scientifique précis, tandis que GMT est une désignation de fuseau horaire qui ne tient pas compte de l'heure d'été.

Qu'est-ce que le Windows FILETIME ?

Le FILETIME Windows est un entier 64 bits comptant les intervalles de 100 nanosecondes depuis le 1er janvier 1601. Il est utilisé dans toute l'API Windows, .NET DateTime et LDAP. Conversion depuis des secondes Unix : FILETIME = (unixSeconds + 11 644 473 600) × 10 000 000.

Comment convertir un timestamp Unix en date Excel ?

Excel compte les jours depuis le 1er janvier 1900. Formule pour les secondes : = A1 / 86400 + 25569. Formatez le résultat en cellule de date. Pour les millisecondes : = A1 / 86400000 + 25569. Cet outil affiche la valeur série Excel dans le tableau des formats de sortie.
