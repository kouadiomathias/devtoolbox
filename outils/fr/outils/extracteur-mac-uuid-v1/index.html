Ce n'est pas théorique. En 2014, la vulnérabilité Heartbleed (CVE‑2014‑0160) a permis à des attaquants d'extraire de la mémoire des serveurs OpenSSL vulnérables. Parmi les données ayant fuité se trouvaient des identifiants UUID v1 qui exposaient directement les adresses MAC exactes des machines de production. Ces adresses ont été utilisées pour cartographier la topologie du réseau interne. Le risque d'exposer des UUID v1 est réel et documenté. Cet outil vous montre exactement ce que vos UUID révèlent — et l'article ci‑dessous en explique les implications en matière de sécurité, de vie privée et de réglementation.

Structure d'un UUID v1 — Bit par bit

Un UUID v1 est une valeur de 128 bits organisée de manière très précise. Comprendre cette disposition est indispensable pour extraire l'horodatage et l'adresse MAC.

6ba7b810-9dad-11d1-80b4-00c04fd430c8
├────────────── Horodatage 60 bits ──────────────┤ │v1│ ├─ Séq. horloge ─┤ ├────── Nœud 48 bits (adresse MAC) ──────┤
• 60 bits : horodatage (intervalles de 100 ns depuis le 15 octobre 1582)
• 4 bits : quartet de version (0001 = 1)
• 14 bits : séquence d'horloge
• 2 bits : variante (10)
• 48 bits : nœud = adresse MAC (ou aléatoire si le bit multicast = 1)

Les 60 bits de l'horodatage ne sont pas stockés de manière séquentielle. Ils sont répartis dans trois champs — time_low (32 bits), time_mid (16 bits) et time_hi_and_version (12 bits après retrait des 4 bits de version). Pour reconstituer la valeur originale de 60 bits, il faut les réassembler dans le bon ordre : time_hi (12 bits de poids fort), puis time_mid (16 bits du milieu), puis time_low (32 bits de poids faible). Cette disposition contre‑intuitive piège de nombreux développeurs.

Code pour tout extraire — Python, JavaScript, Java

Les fonctions suivantes extraient l'horodatage, l'adresse MAC et détectent si le nœud correspond à une véritable adresse matérielle ou à une valeur aléatoire.

Python

import uuid
from datetime import datetime, timezone

def extraire_infos_v1(chaine_uuid: str) -> dict:
    u = uuid.UUID(chaine_uuid)
    if u.version != 1:
        raise ValueError(f"UUID version {u.version} — seul v1 contient un horodatage et une MAC")
    
    DECALAGE_GREGORIEN = 122192928000000000  # intervalles de 100 ns entre le 15/10/1582 et l'epoch Unix
    ts_unix = (u.time - DECALAGE_GREGORIEN) / 1e7
    date_creation = datetime.fromtimestamp(ts_unix, tz=timezone.utc)
    
    mac_int = u.node
    mac = ':'.join(f'{(mac_int >> (5-i)*8) & 0xff:02x}' for i in range(6))
    
    # Bit multicast : si le bit de poids faible du premier octet est 1,
    # le nœud a été généré aléatoirement (pas une vraie MAC)
    est_mac_aleatoire = bool((mac_int >> 40) & 1)
    
    return {
        "version": 1,
        "horodatage": date_creation.isoformat(),
        "adresse_mac": mac,
        "mac_aleatoire": est_mac_aleatoire,
        "sequence_horloge": u.clock_seq
    }

# Exemple
resultat = extraire_infos_v1("6ba7b810-9dad-11d1-80b4-00c04fd430c8")
print(resultat)

#   "adresse_mac": "00:c0:4f:d4:30:c8",
#   "mac_aleatoire": False
# }

JavaScript (navigateur ou Node)

function extraireInfosV1(chaineUUID) {
  const parties = chaineUUID.replace(/-/g, '');
  const version = parseInt(parties[12], 16);
  if (version !== 1) throw new Error(`UUID v${version} — seul v1 contient horodatage et MAC`);

  // Réassembler l'horodatage : time_hi (12 bits), time_mid (16 bits), time_low (32 bits)
  const timeHi  = parseInt(parties.slice(12, 16), 16) & 0x0fff;
  const timeMid  = parseInt(parties.slice(8,  12), 16);
  const timeLow  = parseInt(parties.slice(0,   8), 16);
  const timestamp = BigInt(timeHi) * 281474976710656n +
                    BigInt(timeMid) * 4294967296n +
                    BigInt(timeLow);

  const DECALAGE_GREGORIEN = 122192928000000000n;
  const unixMs = Number((timestamp - DECALAGE_GREGORIEN) / 10000n);

  // Extraire la MAC (12 derniers caractères hexa)
  const macHex = parties.slice(20);
  const mac = macHex.match(/../g).join(':');
  const estMacAleatoire = (parseInt(macHex.slice(0,2), 16) & 0x01) === 1;

  return {
    horodatage: new Date(unixMs).toISOString(),
    adresse_mac: mac,
    mac_aleatoire: estMacAleatoire,
    sequence_horloge: parseInt(parties.slice(16, 20), 16) & 0x3fff
  };
}

Java

import java.util.UUID;
import java.time.Instant;

public class ExtracteurV1 {
    private static final long DECALAGE_GREGORIEN = 0x01b21dd213814000L; // 122192928000000000 * 100ns

    public static void extraire(UUID uuid) {
        if (uuid.version() != 1) {
            throw new IllegalArgumentException("Ce n'est pas un UUID v1");
        }
        
        // Horodatage en intervalles de 100 ns depuis le 15 octobre 1582
        long timestamp = uuid.timestamp();
        long unixMillis = (timestamp - DECALAGE_GREGORIEN) / 10_000L;
        Instant instant = Instant.ofEpochMilli(unixMillis);
        
        // Nœud = adresse MAC (48 bits)
        long noeud = uuid.node();
        StringBuilder mac = new StringBuilder();
        for (int i = 5; i >= 0; i--) {
            int octet = (int) ((noeud >>> (i * 8)) & 0xff);
            if (mac.length() > 0) mac.append(':');
            mac.append(String.format("%02x", octet));
        }
        
        boolean estMacAleatoire = ((noeud >>> 40) & 1) == 1;
        
        System.out.println("Horodatage : " + instant);
        System.out.println("MAC : " + mac);
        System.out.println("MAC aléatoire : " + estMacAleatoire);
    }
}

RGPD et données personnelles — Une préoccupation réelle

Au sens du RGPD, l'article 4 définit les données à caractère personnel comme toute information relative à une personne physique identifiable. Une adresse MAC est liée à un appareil et, dans de nombreux contextes, à une personne. La CNIL a systématiquement considéré les adresses MAC comme des données personnelles dans plusieurs décisions, notamment concernant le suivi Wi‑Fi et le fingerprinting d'appareils.

Si votre application génère des UUID v1 et les expose dans des APIs publiques, des logs ou des exports, vous traitez peut‑être des données personnelles sans le savoir. L'adresse MAC intégrée peut relier des actions à une machine spécifique et, dans un environnement professionnel, à un employé. Consultez votre DPO et envisagez de migrer vers UUID v4 (purement aléatoire) ou UUID v7 (horodatage + aléatoire, sans MAC).

Le cas limite critique — Quand la MAC est aléatoire

Depuis la RFC 4122, Section 4.5, les implémentations sont autorisées à générer un nœud aléatoire de 48 bits au lieu d'utiliser la véritable adresse MAC. Python le fait par défaut depuis Python 3.7. La méthode UUID.randomUUID() de Java génère du v4, pas du v1. La détection est simple : si le bit de poids faible du premier octet (le bit multicast) est à 1, le nœud a été généré aléatoirement et ne correspond pas à un appareil physique.

Les fonctions de code ci‑dessus implémentent déjà cette vérification via mac_aleatoire. Notre outil intégré l'affiche également clairement dans les résultats — vous verrez soit "Véritable adresse MAC", soit "Nœud aléatoire (pas une vraie MAC)".

Foire Aux Questions

Puis‑je également extraire l'horodatage d'un UUID v1 ?

Oui. Les 60 premiers bits encodent la date de création avec une précision de 100 nanosecondes. L'outil ci‑dessus et notre décodeur UUID complet l'affichent.

UUID v1 est‑il déprécié ?

La RFC 9562 (mai 2024) recommande UUID v7 plutôt que v1 pour les nouvelles applications. v1 reste valide mais ses caractéristiques de confidentialité le rendent inadapté aux identifiants exposés au public.
