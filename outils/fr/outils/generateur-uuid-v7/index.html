Qu’est‑ce qu’un UUID v7 et pourquoi l’utiliser ?

Un UUID v7 (Universally Unique Identifier version 7) est un identifiant de 128 bits conçu pour être triable dans le temps. Contrairement à l’UUID v4 (aléatoire), il intègre un horodatage Unix en millisecondes sur les 48 premiers bits. Les bits restants sont remplis aléatoirement, garantissant l’unicité même en cas de génération massive dans la même milliseconde.

Cette propriété de tri est cruciale pour les bases de données modernes : un index sur une colonne UUID v7 reste compact et performant, car les nouvelles lignes s’ajoutent toujours à la fin de l’index. Avec un UUID v4, l’ordre aléatoire provoque une fragmentation et ralentit les écritures.

Comment fonctionne l’UUID v7 ?

La spécification (RFC 9562) définit la structure suivante :

  • 48 bits : horodatage Unix en millisecondes (bits 0‑47).
  • 4 bits : version (fixée à 0111, soit 7).
  • 12 bits : aléatoires (bits 48‑59).
  • 2 bits : variant (10), indiquant le format UUID standard.
  • 62 bits : aléatoires (bits 62‑127).

Résultat : chaque UUID v7 contient la date et l’heure exacte de sa création, à la milliseconde près, ce qui permet de trier les identifiants chronologiquement tout en conservant une excellente unicité.

Foire Aux Questions

Quelle est la différence entre UUID v4 et UUID v7 ?

L’UUID v4 est entièrement aléatoire et ne peut pas être trié dans le temps. L’UUID v7 commence par un horodatage, ce qui permet un tri chronologique naturel et améliore les performances des index de bases de données.

Un UUID v7 est‑il vraiment unique ?

Oui. La partie aléatoire de 74 bits (dont 62 après le variant) offre suffisamment de combinaisons pour éviter les collisions, même en cas de génération massive. La probabilité de collision est infime.

Puis‑je générer un UUID v7 en Python ?

Absolument. La bibliothèque standard uuid ne supporte pas encore nativement l’UUID v7 (avant Python 3.14), mais vous pouvez utiliser le module tiers uuid7 ou implémenter la logique vous‑même en combinant time.time_ns() et des octets aléatoires.

Quel est l’avantage par rapport à un ULID ?

L’ULID offre également un tri temporel, mais sa représentation est en base32 (26 caractères). L’UUID v7 reste un UUID standard de 36 caractères, parfaitement compatible avec toutes les bibliothèques UUID existantes.
