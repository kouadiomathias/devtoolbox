Comment utiliser un UUID comme clé primaire dans MySQL

MySQL ne possède pas de type de données UUID natif, mais vous pouvez stocker les UUID dans des colonnes CHAR(36) ou BINARY(16). La fonction UUID() renvoie une chaîne de 36 caractères (avec tirets).

1. Créer une table avec une clé primaire UUID (CHAR(36))

CREATE TABLE utilisateurs (
    id CHAR(36) PRIMARY KEY DEFAULT (UUID()),
    nom VARCHAR(100) NOT NULL,
    email VARCHAR(255) UNIQUE NOT NULL,
    cree_le TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

Ceci utilise les valeurs par défaut d’expression disponibles depuis MySQL 8.0.13. Pour les versions antérieures, utilisez un déclencheur.

2. Insérer des lignes — l’UUID est généré automatiquement

INSERT INTO utilisateurs (nom, email)
VALUES ('Alice', 'alice@exemple.fr');
SELECT * FROM utilisateurs;

3. Utiliser un déclencheur pour les anciennes versions de MySQL

CREATE TRIGGER before_insert_utilisateurs
BEFORE INSERT ON utilisateurs
FOR EACH ROW
SET NEW.id = UUID();

4. Stocker en BINARY(16) pour de meilleures performances

Vous pouvez stocker les UUID dans une colonne BINARY(16) pour réduire l’empreinte :

CREATE TABLE utilisateurs_bin (
    id BINARY(16) PRIMARY KEY DEFAULT (UUID_TO_BIN(UUID())),
    nom VARCHAR(100) NOT NULL
);

Pourquoi utiliser UUID plutôt qu’AUTO_INCREMENT ?

  • Systèmes distribués — les UUID peuvent être générés sur plusieurs nœuds sans collision.
  • Sécurité — les UUID ne révèlent pas le nombre d’enregistrements.
  • Portabilité — les UUID sont plus faciles à fusionner entre bases de données.

Foire Aux Questions

Quelle version d’UUID produit UUID() ?

La fonction UUID() de MySQL génère un UUID v1 (basé sur l’horodatage) qui peut exposer l’adresse MAC. Pour des UUID aléatoires (v4), utilisez UUID_TO_BIN(UUID()) avec un déclencheur ou le code applicatif.

Une clé primaire UUID est‑elle plus lente qu’un auto‑incrément dans MySQL ?

Les UUID sont plus volumineux et peuvent entraîner une fragmentation d’index, surtout avec des UUID aléatoires. Pour InnoDB, préférez BINARY(16) et triez par une colonne d’horodatage pour limiter les divisions de pages.

Puis‑je utiliser UUID() comme valeur par défaut dans MySQL ?

Oui, à partir de MySQL 8.0.13, vous pouvez utiliser DEFAULT (UUID()) pour les colonnes CHAR(36). Pour les versions antérieures, un déclencheur est nécessaire.
