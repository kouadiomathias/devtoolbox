Comment utiliser un UUID comme clé primaire dans PostgreSQL

PostgreSQL prend en charge le type de données uuid de manière native. Vous pouvez générer des valeurs UUID avec la fonction intégrée gen_random_uuid() (disponible depuis PostgreSQL 13) ou via l’extension pgcrypto pour les versions plus anciennes.

1. Activer la fonction (si nécessaire)

Pour PostgreSQL 13+, gen_random_uuid() est disponible par défaut. Pour les versions antérieures, activez l’extension pgcrypto :

CREATE EXTENSION IF NOT EXISTS pgcrypto;

Après cela, gen_random_uuid() sera utilisable.

2. Créer une table avec une clé primaire UUID

CREATE TABLE utilisateurs (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    nom VARCHAR(100) NOT NULL,
    email VARCHAR(255) UNIQUE NOT NULL,
    cree_le TIMESTAMPTZ DEFAULT NOW()
);

3. Insérer des lignes — l’UUID est généré automatiquement

INSERT INTO utilisateurs (nom, email)
VALUES ('Alice', 'alice@exemple.fr')
RETURNING id;

Pourquoi utiliser UUID plutôt que SERIAL ?

  • Systèmes distribués — chaque nœud peut générer des UUID indépendamment sans collision.
  • Sécurité — les UUID ne révèlent pas le nombre d’enregistrements.
  • Pas de goulet d’étranglement séquentiel — contrairement à l’auto‑incrément, les UUID ne créent pas de contention sur la dernière page de l’index.
  • Tri — utilisez UUID v7 si vous avez besoin d’identifiants ordonnés dans le temps (voir notre générateur UUID v7).

Foire Aux Questions

Quelle version d’UUID produit gen_random_uuid() ?

Elle génère un UUID v4 (aléatoire). Si vous avez besoin d’UUID triables, utilisez notre générateur UUID v7 et stockez la valeur manuellement.

Une clé primaire UUID est‑elle plus lente qu’un SERIAL ?

Les UUID sont plus volumineux (16 octets contre 4/8 octets) et peuvent entraîner une fragmentation d’index si vous utilisez des UUID v4 aléatoires. L’UUID v7 ou un UUID séquentiel atténue ce problème.

Puis‑je utiliser gen_random_uuid() comme valeur par défaut ?

Oui, comme indiqué ci‑dessus. La fonction est stable et peut être utilisée directement dans les expressions de colonne DEFAULT.
