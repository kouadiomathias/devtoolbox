Ce que l'entropie de Shannon mesure dans un UUID

L'entropie de Shannon quantifie la quantité d'information — ou d'aléatoire — dans une chaîne. Pour un UUID v4 parfaitement aléatoire, chaque caractère hexadécimal (0‑f) devrait être équiprobable, ce qui donne une entropie attendue de log₂(16) = 4 bits par caractère. En pratique, un UUID v4 sain produit environ 3,95 bits/car parce que les bits de version et de variante biaisent légèrement deux quartets.

Lorsque l'entropie descend en dessous de 3,9, les bits ne sont plus uniformément répartis. C'est la signature d'un générateur de nombres pseudo‑aléatoires (PRNG) faible — exactement ce qui s'est produit avec le bug de Math.random() de V8 en 2012, où l'état interne pouvait être reconstitué après avoir observé seulement quelques sorties.

Analyse par lot — Comment auditer un ensemble d'UUID

Le vérificateur ci‑dessus accepte de coller plusieurs UUID (un par ligne). En mode lot, il calcule :

  • Entropie par position : entropie de chacune des 32 positions hexadécimales sur l'ensemble des UUID. Les positions fixes de version/variante doivent montrer une entropie proche de zéro.
  • Statistique du chi‑carré : teste si la distribution globale des quartets correspond à une distribution uniforme. Une valeur élevée du chi‑carré indique un biais.

L'analyse s'inspire du code Python ci‑dessous, que vous pouvez utiliser sur vos propres données.

Implémentation en Python

import math
from collections import Counter

def analyse_entropie_uuid(uuids: list[str]) -> dict:
    propres = [u.replace('-', '').lower() for u in uuids]
    
    # 1. Entropie de Shannon par position
    entropie_position = []
    for pos in range(32):
        caracteres_pos = [u[pos] for u in propres if len(u) == 32]
        compteurs = Counter(caracteres_pos)
        total = len(caracteres_pos)
        entropie = -sum((c/total) * math.log2(c/total) for c in compteurs.values())
        entropie_position.append(round(entropie, 3))
    
    # 2. Positions fixes (entropie < 0,5 -> constante)
    positions_fixes = [i for i, e in enumerate(entropie_position) if e < 0.5]
    
    # 3. Entropie globale
    tous_caracteres = ''.join(propres)
    compteurs = Counter(tous_caracteres)
    total = len(tous_caracteres)
    entropie_globale = -sum((c/total) * math.log2(c/total) for c in compteurs.values())
    
    # 4. Test du chi‑carré (distribution uniforme attendue)
    freq_attendue = total / 16
    chi_carre = sum((compteurs.get(c, 0) - freq_attendue)**2 / freq_attendue 
                     for c in '0123456789abcdef')
    
    return {
        "entropie_globale_bits": round(entropie_globale, 3),
        "attendu_pour_v4": 3.95,
        "entropie_position": entropie_position,
        "positions_fixes": positions_fixes,
        "chi_carre": round(chi_carre, 2),
        "verdict": "OK" if entropie_globale > 3.8 else "SUSPECT",
        "taille_echantillon": len(uuids)
    }

Détecter un PRNG faible à partir d'un lot d'UUID

Si vous disposez d'un ensemble d'UUID supposés être en v4 et que certaines positions présentent une entropie anormalement basse, vous êtes probablement face à un PRNG faible ou à une graine réutilisée. Un exemple concret : les premières versions du framework PHP Laravel (avant la v5.5) utilisaient mt_rand() ensemencé avec l'horodatage courant, rendant la génération d'UUID prévisible dans une fenêtre de temps étroite.

Passez votre lot dans le vérificateur ci‑dessus — si la valeur du chi‑carré dépasse 30 pour un échantillon de plus de 100 UUID, ou si plusieurs positions affichent une entropie inférieure à 3,5, votre générateur doit être remplacé. Passez à une source cryptographiquement sûre comme random_bytes() en PHP ou crypto.randomUUID() en JavaScript moderne.

Foire Aux Questions

De combien d'UUID ai‑je besoin pour une analyse par lot fiable ?

Un minimum de 50 UUID est recommandé. Le test du chi‑carré devient statistiquement significatif avec des échantillons plus grands (plus de 100).

Cet outil peut‑il détecter un PRNG faible à partir d'un seul UUID ?

Avec un seul UUID, nous pouvons mesurer l'entropie de Shannon de la chaîne et signaler des valeurs anormalement basses. Cependant, l'analyse par lot est bien plus fiable pour détecter des distributions non uniformes.

Pourquoi mon entropie est‑elle légèrement inférieure à 4 bits/car ?

L'UUID v4 réserve 6 bits pour les marqueurs de version et de variante. Ces bits sont fixes, donc deux caractères hexadécimaux ne peuvent pas être entièrement aléatoires. Le maximum théorique est d'environ 3,98 bits/car.
