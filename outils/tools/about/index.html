Our story

How DevToolbox came to be

It started with the same frustration most developers have felt at least once: you need to quickly format a JSON response, or decode a JWT to check what's inside, and every tool you find either loads slowly because of advertising scripts, shows you a wall of ads before you can even paste anything, or — worse — isn't clear about whether your data is processed on their server or in your browser.

Those aren't small concerns. We regularly work with API payloads that contain authentication tokens, internal user identifiers, and configuration data that should never leave our machines. The answer to "is this tool client-side?" shouldn't require reading fine print.

"The answer to 'is this client-side?' shouldn't require reading fine print."

So we started building the tools we actually wanted. Fast pages with no advertising network loading in the background. Explicit documentation of what stays in the browser. An interface that respects your attention during a debugging session rather than fighting it for ad impressions.

We're a small team. DevToolbox isn't a venture-backed product — it's a project we maintain because we use it ourselves, and because we think the developer tooling space deserves better defaults around privacy and interface quality. We publish in English and French, and we're growing the catalog based on what actually comes up in daily work rather than trying to reach an arbitrary number of tools.

If you've used DevToolbox and found something that was wrong, slow, or missing — we want to hear it. The project improves because of that feedback.

What we stand for

Three things we don't compromise on

Privacy by default

Every tool on DevToolbox documents exactly where processing happens. If a tool runs in your browser, we say so explicitly — and it means your data genuinely doesn't touch a server. We don't make you trust us; we make trust unnecessary.

Respect for your time

A tool that takes three seconds to load because it's serving advertising to you is a tool that's working against you. We keep pages light, interfaces clean, and interactions immediate. No banners. No modal sign-up prompts. No dark patterns.

Quality over quantity

We'd rather have 30 tools that work exceptionally well than 200 that are mediocre. Every tool we add is one we've found ourselves reaching for. If it doesn't make the daily workflow better, it doesn't belong here.
