Base64 Encoding in Practice: Where It Breaks and How to Fix It

On this page
  1. The InvalidCharacterError nobody expected
  2. Quick decision table
  3. Common Base64 pitfalls
  4. FAQ
  5. References

A translated-content pipeline worked flawlessly in staging, where every test string was plain English. In production, the first French payload containing an accented character — café — hit btoa() and the whole request crashed with InvalidCharacterError: Failed to execute 'btoa' on 'Window'. btoa() only accepts Latin1 (single-byte) characters; any string with an accented letter, an emoji, or any non-ASCII character throws immediately. The fix wasn't to avoid accents — it was to encode the string's actual UTF-8 bytes instead of its raw JavaScript characters.

That single gotcha causes a disproportionate number of Base64-related bug reports, but it's far from the only one. Base64 looks trivial — three bytes in, four characters out — yet the surrounding details (URL-safety, padding, encoding, storage) are where real production issues show up.

Base64 is not encryption. It has no key and no secret — it's a reversible text encoding. Never use it to protect passwords, tokens, or PII. Anyone who receives the string can decode it in one line.

Common Base64 pitfalls

01btoa() throwing on non-ASCII text
High

btoa() operates on UTF-16 code units and requires every one to fit in a single byte (0–255). Any accented letter, non-Latin script, or emoji throws InvalidCharacterError immediately.

Avoid
const b64 = btoa("café"); // throws InvalidCharacterError
Fix
function encodeUtf8Base64(str) { const bytes = new TextEncoder().encode(str); let binary = ''; bytes.forEach(b => binary += String.fromCharCode(b)); return btoa(binary); // works for any valid Unicode string }
02Standard Base64 breaking inside a URL
High

+ is interpreted as a space in query strings, and / looks like a path separator. A Base64 string containing either will silently corrupt the URL it's embedded in.

Avoid
https://api.example.com/verify?token=a+b/c== // '+' and '/' get mangled by URL parsing
Fix
b64url = b64.replace(/\+/g, '-').replace(/\//g, '_').replace(/=+$/, ''); // this is exactly what the URL-safe toggle above does
03Forgetting to strip the Data URL prefix
Medium

A full Data URL looks like data:image/png;base64,iVBORw0KG.... Passing that entire string to atob() or base64.b64decode() fails or produces garbage, because the metadata prefix isn't valid Base64.

Fix
const raw = dataUrl.split(',')[1]; // drop everything before the first comma const bytes = atob(raw);
04Storing Base64 blobs in relational columns
Medium

Base64 already adds ~33% overhead over the raw bytes. Storing that text in a TEXT/VARCHAR column instead of native binary adds further overhead from string collation, indexing, and query planning that binary types avoid.

Fix
-- PostgreSQL: store raw bytes, not Base64 text CREATE TABLE attachments (id uuid PRIMARY KEY, data bytea); -- or better: store the file in object storage (S3, GCS) and keep only a URL/key

Frequently asked questions

Is Base64 encryption?
No. Base64 is a fully reversible encoding with no key. Anyone who has the string can decode it instantly. Use real encryption (AES, RSA) for anything that needs to stay secret.
What is the difference between Base64 and Base64url?
Base64url replaces the two URL-unsafe characters — + becomes -, / becomes _ — and drops the = padding, so the result is safe inside URLs, filenames, and HTTP headers. JWT header and payload segments always use Base64url.
How do I convert an image or file to Base64 in the browser?
Drop a file on the "File / Image → Base64" panel above, or in your own code use the FileReader API's readAsDataURL() method — it returns a ready-to-use Data URL.
How do I encode Base64 in JavaScript without it breaking on accents?
Don't call btoa() directly on a Unicode string. Encode it to UTF-8 bytes first with TextEncoder, convert those bytes to a binary string, then call btoa() on that. Decoding is the same process in reverse with atob() and TextDecoder.
How do I encode Base64 in Python?
Use the base64 module: base64.b64encode(b"Hello"). For URL-safe output, use base64.urlsafe_b64encode(b"Hello"). Both accept and return bytes, so encode/decode strings with .encode()/.decode() around them.
Why does Base64 sometimes end with == ?
Base64 processes input 3 bytes at a time into 4 characters. When the input length isn't a multiple of 3, = padding is added so the output length is a multiple of 4 — one = for 1 byte short, two == for 2 bytes short. Base64url omits this padding entirely.
Is my data sent to a server when I use this tool?
No. All encoding, decoding, and file processing happens entirely in your browser via JavaScript. Nothing you type, paste, or drop here ever leaves your machine.
