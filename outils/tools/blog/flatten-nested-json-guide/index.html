What does “flatten nested JSON” actually mean?

It means taking a structure like this: 

[
  {
    "name": "Alice",
    "address": {
      "city": "Paris",
      "zip": "75001"
    }
  }
]

And turning it into a flat table: 

name,address.city,address.zip
Alice,Paris,75001

The address sub‑object disappears; its fields become new columns with dot‑notation headers. That’s JSON to Excel online 101, and it’s exactly what our JSON to CSV Converter does automatically when you check the Flatten objects option.

Why flatten JSON? Real‑world reasons

  • Convert API JSON to CSV for business reports. Your backend sends nested data; your stakeholder wants a table.
  • Prepare data for databases. Most SQL tools import flat CSV better than deeply nested JSON.
  • Debug JWT tokens. Often a JWT payload contains nested claims. You can decode the JWT, then flatten its payload into CSV for analysis.
  • Share with non‑tech colleagues. They know Excel, not JSON paths.

How flattening works under the hood

Flattening a JSON object is a recursive process. Starting from the top level, we walk through every key. If the value is another object (and not an array), we go deeper and prepend the parent key to the child keys, separating them with a dot. If the value is a simple type (string, number, boolean, null), we store it as a column.

Here's the core algorithm in Python — just a few lines: 

def flatten(obj, prefix='', sep='.'):
    items = {}
    for key, val in obj.items():
        new_key = f"{prefix}{sep}{key}" if prefix else key
        if isinstance(val, dict):
            items.update(flatten(val, new_key, sep))
        else:
            items[new_key] = val
    return items

And the same logic in JavaScript, ready to run in your browser: 

function flatten(obj, prefix = '', sep = '.') {
  return Object.keys(obj).reduce((acc, key) => {
    const newKey = prefix ? `${prefix}${sep}${key}` : key;
    if (typeof obj[key] === 'object' && !Array.isArray(obj[key]) && obj[key] !== null) {
      Object.assign(acc, flatten(obj[key], newKey, sep));
    } else {
      acc[newKey] = obj[key];
    }
    return acc;
  }, {});
}

When you apply this to an array of objects (like an API response), you first flatten each element, then combine all keys into the CSV header. That's exactly what the Devstoolsbox converter does in real‑time.

Edge cases that surprise developers

Flattening is simple until you hit one of these real‑world scenarios:

Arrays of objects

If a field contains an array of objects — say orders: [{id: 1}, {id: 2}] — the basic flatten function can't turn that into a single column. Our tool outputs each array element as a new row, duplicating the parent fields. That's often what you want for a flat table, but it can lead to many rows. If you'd rather keep the array as a JSON string inside the cell, you can adjust the flattening options in the converter.

Null values

What happens when a field is null? In Devstoolsbox, null becomes an empty cell. This is clean for Excel; other tools might drop the column entirely. You can also choose to represent nulls as the string "null" or as 0 via the empty‑cell handling settings.

Dots in original key names

If your JSON already has keys like "first.name", flattening with a dot separator will create a collision. Our tool warns you when it detects dots inside keys. You can switch the separator to __ or rename those keys before flattening.

Maximum depth

While there's no hard limit, extremely deep nesting (10+ levels) can produce hundreds of columns. In such cases, you might want to flatten only the first few levels and keep the rest as a serialized JSON string. The converter lets you set a maximum flatten depth.

Flatten vs. normalize: which one do you really need?

People often confuse flattening with normalization. They serve different purposes.

Flattening produces a single wide table. Every nested object becomes extra columns; every array element becomes an extra row with parent data repeated. This is great for quick exports, Excel analysis, and reports.

Normalizing splits the data into multiple related tables — for example, a users.csv and an orders.csv linked by a user_id. This is the clean approach for relational databases.

When you flatten a user with orders, you get: 

name,order.id,order.amount
Alice,1,50
Alice,2,30

If you normalize, you'd export two CSVs: 

# users.csv
id,name
1,Alice

# orders.csv
user_id,id,amount
1,1,50
1,2,30

Which one you pick depends on the downstream tool. Excel users love a flat table; database administrators prefer normalized tables.

For large files: pandas and jq

The online tool shines for quick, interactive flattening. But when you're dealing with a multi‑gigabyte JSON export, you'll want a command‑line approach. Here are two proven methods.

Python with pandas

Data analysts love pd.json_normalize. It does exactly what our tool does, but in a scriptable way. This snippet reads a JSON file, flattens the array at data.users, and writes a CSV with proper UTF‑8 BOM for Excel: 

import pandas as pd
import json

with open('data.json') as f:
    data = json.load(f)

df = pd.json_normalize(data['data']['users'], sep='.')
df.to_csv('output.csv', index=False, encoding='utf-8-sig')

utf-8-sig is Python's way of adding the UTF‑8 BOM. That little detail saves hours when colleagues open the file in Excel.

jq for the terminal

jq is a lightweight and incredibly fast JSON processor. To extract and flatten a subset of fields from a large nested array, you can run: 

jq -r '.data.users[] | [.name, .address.city, .address.zip] | @csv' data.json

This pipes the flattened rows directly to stdout, which you can redirect to a CSV file. It's ideal for automated pipelines.

Frequently Asked Questions

Why does my flattened CSV contain repeated rows for the same user?

That happens when you flatten a JSON object that contains an array (like orders). Each element of the array creates a new row with the parent columns repeated. If you want to avoid duplication, you can either export multiple CSV files (normalize) or decide how you want arrays to be represented (as JSON string, only first element, etc.).

What is the difference between flattening and normalizing JSON?

Flattening creates a single wide table; normalizing splits the data into several related tables. Use flattening for quick export, normalizing for database imports.

Can I flatten JSON that has dots in the key names?

It's risky because dots are used as separators. Our tool detects dots in original keys and warns you. You can choose a different separator (like __) or rename the keys before flattening.

What happens to null values when I flatten JSON?

In our tool, null values become empty cells. That's usually what you want. If you need explicit 'null' strings or zeros, adjust the empty cell handling in the converter settings.

How can I flatten extremely large JSON files (gigabytes)?

Use command‑line tools like jq or Python with pandas. We show examples in the article. Our online tool works best up to a few megabytes of JSON, completely in your browser.
