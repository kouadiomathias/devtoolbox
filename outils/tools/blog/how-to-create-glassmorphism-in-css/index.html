How to Create Glassmorphism in CSS – Real‑World Examples and Pro Tips

Published on April 28, 2026 · Updated May 18, 2026 — 10 min read

Glassmorphism has taken over dashboards, landing pages, and mobile interfaces. Those frosted glass panels that blur the background just enough to create depth — they're built with pure CSS, no extra plugins. In this guide, we’ll build a glass card step by step, then dig into the details that trip up most developers: the difference between backdrop-filter and filter, performance on mobile, stacking context surprises, and how to keep the effect accessible. Along the way, you’ll find ready‑to‑use code snippets and a link to our free visual generator that does the heavy lifting for you.

The three ingredients of glassmorphism

A convincing glass effect always relies on:

  • A semi‑transparent background, usually white or a very light tint.
  • A backdrop‑filter blur that freezes the content behind the element.
  • A subtle border that defines the edge of the glass.

The basic glass card CSS

Here’s the shortest version that works across modern browsers: 

.glass-card {
  background: rgba(255, 255, 255, 0.2);
  backdrop-filter: blur(12px);
  -webkit-backdrop-filter: blur(12px); /* Safari */
  border-radius: 20px;
  border: 1px solid rgba(255, 255, 255, 0.3);
  padding: 28px;
  color: #fff;
}

Let’s break down each property. background: rgba(255,255,255,0.2) gives you that almost‑invisible white layer. The alpha value controls how much the background peeks through. backdrop-filter: blur(12px) does the actual blurring — it’s a Gaussian blur applied to everything behind the element. The -webkit- prefix is still needed for Safari. Finally, the border adds a faint glassy rim; without it, the card can melt into the background.

backdrop-filter vs filter: a crucial distinction

Many newcomers confuse backdrop-filter with filter. They’re fundamentally different, and using the wrong one breaks the glass effect entirely.

  • filter applies to the element itself and all its children. If you write filter: blur(5px), the card and the text inside it will look like smudged glass — not what you want.
  • backdrop-filter only affects what lies behind the element. The card itself stays sharp; only the background gets the frost treatment.

A side‑by‑side comparison makes it clear:

/* ❌ wrong for glass: the whole card becomes blurry */
.card-blurry {
  filter: blur(8px);
}

/* ✅ correct: only the background is blurred */
.card-glass {
  backdrop-filter: blur(12px);
}

So remember: backdrop-filter for glass, filter for manipulating the element itself (like grayscale or opacity changes).

Performance: when blur hurts frame rate

backdrop-filter forces the browser to create a new composite layer. That layer gets redrawn every time the background changes, which can be costly on mobile devices or when animated. A heavy blur (20px+) on a large card can drop frames, especially on budget phones.

Practical rules for smooth glass effects:

  • Keep blur under 12-15px for elements that animate or scroll. Static cards can go higher.
  • Use will-change: backdrop-filter sparingly — it tells the browser to prepare a separate layer, but overusing it wastes GPU memory.
  • Always test on a real device, not just Chrome DevTools. Open the Performance tab and look for composite layer repaints.

The stacking context trap (and why z-index: -1 breaks)

A subtle but common bug: backdrop-filter creates a stacking context. That means any element behind the glass card with a negative z-index might get hidden, even if you expect it to stay visible. This happens because the glass element and the negative‑z background now belong to different stacking layers.

For example, if you place a gradient background with z-index: -1 and then a glass card on top, the background might disappear behind the card — not because of the blur, but because of the stacking context. The fix? Avoid negative z‑indexes on backgrounds; instead, place the background in a wrapper with a positive z‑index or rely on natural DOM order.

Glassmorphism vs neumorphism vs skeuomorphism

Glassmorphism is often mentioned alongside neumorphism and skeuomorphism. They all create depth, but each has a different philosophy.

AspectGlassmorphismNeumorphismSkeuomorphism
Visual styleFrosted glass, transparencySoft extruded shapes, monochrome shadowsRealistic textures (leather, wood, metal)
Key techniquebackdrop-filterMultiple box‑shadowsGradients, textures, complex images
ReadabilityGood if contrast is checkedOften poor; low contrastVariable
PerformanceModerate (composite layer)LightHeavy (images, large CSS)
Best forDashboards, modals, overlaysMinimalist controls, togglesLegacy apps, retro UIs

Use glassmorphism when you want depth but still need to show the background content. Neumorphism suits simple, static controls. Skeuomorphism is mostly historical now.

Tailwind CSS glass card

If you’re working with Tailwind, you can achieve the same effect with utility classes. Here’s a glass card in pure Tailwind: 

<div class="backdrop-blur-md bg-white/20 border border-white/30 rounded-2xl p-7">
  <h2 class="text-white text-xl font-semibold">Glass Card</h2>
  <p class="text-white/80">Built with Tailwind.</p>
</div>

backdrop-blur-md corresponds to a medium blur (12px by default). You can use backdrop-blur-sm or backdrop-blur-lg as needed.

React component with styled‑components

To make a reusable glass card in a React project, you can create a styled component with adjustable opacity and blur: 

import styled from 'styled-components';

const GlassCard = styled.div`
  background: rgba(255, 255, 255, ${props => props.opacity || 0.2});
  backdrop-filter: blur(${props => props.blur || 12}px);
  -webkit-backdrop-filter: blur(${props => props.blur || 12}px);
  border-radius: 20px;
  border: 1px solid rgba(255, 255, 255, 0.3);
  padding: 28px;
  color: white;
`;

// Usage
<GlassCard opacity={0.25} blur={16}>
  <h2>Dashboard</h2>
</GlassCard>

Progressive fallback for unsupported browsers

Instead of leaving IE11 users with a completely broken layout, provide a solid semi‑opaque background as a fallback. The @supports at‑rule lets you write clean, progressive enhancement: 

.glass-card {
  /* Fallback: solid but translucent */
  background: rgba(255, 255, 255, 0.7);
}

@supports (backdrop-filter: blur(1px)) {
  .glass-card {
    background: rgba(255, 255, 255, 0.2);
    backdrop-filter: blur(12px);
    -webkit-backdrop-filter: blur(12px);
  }
}

This way, every browser gets a usable design. IE11 sees a solid white card; modern browsers get the frosted glass.

Accessibility: making glass readable for everyone

backdrop-filter creates a dynamic contrast problem. The text sits on a background whose brightness depends on what’s behind the glass — a dark image might give enough contrast, while a light gradient might not. Static contrast checkers can’t capture this.

Here’s how to keep your glass panels accessible:

  • Test the card against the lightest possible background it will ever have. If the text passes WCAG AA contrast in that worst‑case scenario, it’s safe.
  • Increase the background opacity slightly (to 0.3–0.4) if readability is weak.
  • Add a subtle text‑shadow to improve legibility, e.g., text-shadow: 0 1px 3px rgba(0,0,0,0.5);

Frequently Asked Questions

Why does my glassmorphism effect break when I use z-index: -1?

backdrop-filter creates a new stacking context. Elements with negative z‑index behind the glass card may get hidden. Instead of negative z‑index, restructure the HTML so the background is naturally behind the card.

Is backdrop-filter heavy on mobile?

It can be. On low‑end devices, a large blur can drop frames. Keep blur under 12px for animated elements, use will-change only when needed, and test on real hardware.

How can I make glassmorphism accessible?

Since contrast is dynamic, test against the lightest background. If needed, increase background opacity or add a subtle text shadow. Always preview on real background images.
