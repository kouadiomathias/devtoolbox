The most common GUID mistake in .NET

These two lines look nearly identical. One of them is a critical bug:

// ❌ WRONG — produces 00000000-0000-0000-0000-000000000000 every time
public Guid Id { get; set; } = new Guid();

// ✅ CORRECT  generates a new unique UUID v4
public Guid Id { get; set; } = Guid.NewGuid();

new Guid() calls the parameterless constructor, which initializes every bit to zero the same as Guid.Empty. Every entity you create gets the same all-zeros primary key. The first database insert succeeds; every subsequent one hits a primary key constraint or silently overwrites the first record.

Frequently asked questions

Is a GUID the same as a UUID in C#?
Yes. GUID is Microsoft's term for the UUID standard defined in RFC 4122. System.Guid.NewGuid() generates a UUID v4. The binary layout is identical.
What is the difference between Guid.NewGuid() and new Guid()?
Guid.NewGuid() generates a cryptographically unique random UUID v4 each time. new Guid() the parameterless constructor — returns Guid.Empty, which is 00000000-0000-0000-0000-000000000000. This is the #1 GUID bug in .NET.
How do I convert a GUID to a string without hyphens?
Call id.ToString("N"). This produces a 32-character lowercase hex string. To parse it back, use Guid.ParseExact(str, "N").
Which GUID format should I use in a JSON REST API?
Format D the default hyphenated form. id.ToString() or id.ToString("D"). Every JSON library in every language can parse it without configuration.
How do I use a GUID as a primary key in Entity Framework Core?
Declare the key property as Guid. EF Core generates a new GUID on insert using ValueGeneratedOnAdd(). For SQL Server, consider HasDefaultValueSql("NEWSEQUENTIALID()") to avoid index fragmentation. On .NET 9+, use Guid.CreateVersion7().
Should I use Guid.NewGuid() or UUID v7 for primary keys in 2026?
For database primary keys, UUID v7 is the better choice: its timestamp prefix keeps values monotonically increasing, eliminating B-tree index fragmentation. Guid.CreateVersion7() is built into .NET 9+. For tokens and external identifiers, stick with Guid.NewGuid() (UUID v4).
