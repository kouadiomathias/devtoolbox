package main

import (
    "fmt"
    "time"
)

func main() {
    // Current Unix timestamp in seconds
    nowSeconds := time.Now().Unix()

    // Current Unix timestamp in nanoseconds
    nowNano := time.Now().UnixNano()

    // Current Unix timestamp in milliseconds
    nowMs := time.Now().UnixMilli() // Go 1.17+

    // Unix seconds → time.Time (UTC)
    t := time.Unix(nowSeconds, 0).UTC()

    // Format as ISO 8601 (RFC3339)
    fmt.Println(t.Format(time.RFC3339))       // "2025-06-08T10:00:00Z"
    fmt.Println(t.Format(time.RFC3339Nano))   // With nanoseconds

    // Format as RFC 2822 (email standard)
    fmt.Println(t.Format(time.RFC1123Z))

    // Timezone-aware
    paris, _ := time.LoadLocation("Europe/Paris")
    tParis := t.In(paris)
    fmt.Println(tParis.Format("2006-01-02 15:04:05 MST"))

    // time.Time → Unix timestamp
    d := time.Date(2025, 6, 8, 10, 0, 0, 0, time.UTC)
    fmt.Println(d.Unix())  // 1749376800

    _ = nowNano
    _ = nowMs
}

What is a Unix timestamp?

A Unix timestamp (also called Unix time or POSIX time) is the number of seconds elapsed since January 1, 1970 at 00:00:00 UTC — a reference point called the Unix epoch. It is timezone-independent, making it the universal standard for storing and comparing dates in databases, APIs, logs, and distributed systems.

Two formats coexist in practice: seconds (a 10-digit number like 1749376800) and milliseconds (a 13-digit number like 1749376800000). JavaScript's Date.now() and Java's System.currentTimeMillis() return milliseconds. Python's time.time() and most Unix system calls return seconds. This tool auto-detects both.

UUID timestamps explained

UUID v1 — timestamp + MAC address

UUID v1 encodes a 60-bit timestamp counting 100-nanosecond intervals since October 15, 1582 (the start of the Gregorian calendar). The timestamp is stored in three non-contiguous fields: time_low (32 bits), time_mid (16 bits), and time_hi (12 bits + version nibble). Reassembling them requires reversing this layout. The UUID v1 decoder above handles this automatically.

UUID v7 — Unix timestamp in the first 48 bits

UUID v7 (RFC 9562, May 2024) uses a much simpler approach: the first 48 bits directly encode the Unix timestamp in milliseconds. This makes UUIDs naturally sortable by creation time, and extraction is trivial — just read the first 12 hex characters and convert from hexadecimal.

The Year 2038 problem

On 32-bit systems, Unix timestamps are stored as signed 32-bit integers with a maximum of 2,147,483,647, corresponding to January 19, 2038 at 03:14:07 UTC. After that point, the value overflows to a large negative number. Most modern systems use 64-bit integers (valid until the year 292,277,026,596), but embedded systems and legacy code may still be affected.

Windows FILETIME conversion

Windows FILETIME is a 64-bit integer counting 100-nanosecond intervals since January 1, 1601. The offset between the Windows epoch and the Unix epoch is exactly 11,644,473,600 seconds (or 116,444,736,000,000,000 in 100ns units).

Conversion: FILETIME = (unixSeconds + 11644473600) × 10,000,000

In .NET: DateTime.FromFileTimeUtc(filetime) or DateTimeOffset.FromUnixTimeSeconds(unixSeconds)

Frequently Asked Questions

What is a Unix timestamp?

A Unix timestamp is the number of seconds elapsed since January 1, 1970 at 00:00:00 UTC. It is timezone-independent and used universally in databases, APIs, and logs to store and compare dates without ambiguity.

How do I tell if a timestamp is in seconds or milliseconds?

A 10-digit number is almost certainly seconds (valid until year 2286). A 13-digit number is milliseconds. This tool auto-detects: values above 10,000,000,000 are treated as milliseconds. When in doubt, check the number of digits.

Can I extract a date from a UUID v1?

Yes. UUID v1 encodes a 60-bit timestamp counting 100-nanosecond intervals since October 15, 1582. The "Special identifier decoders" section above extracts this timestamp and converts it to every date format. It also detects whether the MAC address in the UUID is real or randomly generated.

What is the difference between UTC and GMT?

UTC (Coordinated Universal Time) is the modern international standard maintained by atomic clocks. GMT (Greenwich Mean Time) is a timezone. For most practical purposes they are equivalent — UTC+0 and GMT represent the same time. However, UTC is the precise scientific standard, while GMT is a timezone designation that does not adjust for DST.

What is Windows FILETIME?

Windows FILETIME is a 64-bit integer counting 100-nanosecond intervals since January 1, 1601. It is used throughout the Windows API, .NET DateTime, and LDAP. Convert from Unix seconds: FILETIME = (unixSeconds + 11,644,473,600) × 10,000,000.

How do I convert a Unix timestamp to an Excel date?

Excel counts days since January 1, 1900. Formula for seconds: = A1 / 86400 + 25569. Format the result as a date cell. For milliseconds: = A1 / 86400000 + 25569. This tool shows the Excel serial value in the output formats table.
