What Shannon Entropy Measures in a UUID

Shannon entropy quantifies the amount of information — or randomness — in a string. For a perfectly random UUID v4, every hex character (0‑f) should be equally likely, giving an expected entropy of log₂(16) = 4 bits per character. In practice, a healthy UUID v4 yields about 3.95 bits/char because the version and variant bits slightly bias two nibbles.

When entropy drops below 3.9, the bits are no longer uniformly distributed. This is the signature of a weak pseudo‑random number generator (PRNG) — exactly what happened with the V8 Math.random() bug in 2012, where the internal state could be recovered after observing only a few outputs.

Batch Analysis — How to Audit a Set of UUIDs

The checker above supports pasting multiple UUIDs (one per line). When you switch to batch mode, it computes:

  • Per‑position entropy: entropy of each of the 32 hex positions across all UUIDs. Fixed version/variant positions should show near‑zero entropy.
  • Chi‑square statistic: tests whether the overall nibble distribution matches a uniform distribution. A high chi‑square value indicates bias.

The analysis is inspired by the Python code below, which you can use on your own data.

Python Implementation

import math
from collections import Counter

def uuid_entropy_analysis(uuids: list[str]) -> dict:
    clean = [u.replace('-', '').lower() for u in uuids]
    
    # 1. Shannon entropy per position
    position_entropy = []
    for pos in range(32):
        chars_at_pos = [u[pos] for u in clean if len(u) == 32]
        counts = Counter(chars_at_pos)
        total = len(chars_at_pos)
        entropy = -sum((c/total) * math.log2(c/total) for c in counts.values())
        position_entropy.append(round(entropy, 3))
    
    # 2. Fixed positions (entropy < 0.5 -> constant)
    fixed_positions = [i for i, e in enumerate(position_entropy) if e < 0.5]
    
    # 3. Global entropy
    all_chars = ''.join(clean)
    counts = Counter(all_chars)
    total = len(all_chars)
    global_entropy = -sum((c/total) * math.log2(c/total) for c in counts.values())
    
    # 4. Chi‑square test (expected uniform distribution)
    expected_freq = total / 16
    chi_square = sum((counts.get(c, 0) - expected_freq)**2 / expected_freq 
                     for c in '0123456789abcdef')
    
    return {
        "global_entropy_bits": round(global_entropy, 3),
        "expected_for_v4": 3.95,
        "position_entropy": position_entropy,
        "fixed_positions": fixed_positions,
        "chi_square": round(chi_square, 2),
        "verdict": "OK" if global_entropy > 3.8 else "SUSPECT",
        "sample_size": len(uuids)
    }

Detecting a Weak PRNG from a Batch of UUIDs

If you have a set of supposedly v4 UUIDs and certain positions show abnormally low entropy, you're likely looking at a weak PRNG or a reused seed. A concrete example: early versions of the PHP Laravel framework (before v5.5) used mt_rand() seeded with the current timestamp, making UUID generation predictable within a small time window.

Run your batch through the checker above — if the chi‑square value exceeds 30 for a sample of 100+ UUIDs, or if multiple positions show entropy below 3.5, your generator needs replacing. Switch to a cryptographically secure source like random_bytes() in PHP or crypto.randomUUID() in modern JavaScript.

Frequently Asked Questions

How many UUIDs do I need for reliable batch analysis?

A minimum of 50 UUIDs is recommended. The chi‑square test becomes statistically meaningful with larger samples (100+).

Can this tool detect a weak PRNG from a single UUID?

With a single UUID, we can measure the Shannon entropy of the string and flag suspiciously low values. However, batch analysis is far more reliable for detecting non‑uniform distributions.

Why is my entropy slightly below 4 bits/char?

UUID v4 reserves 6 bits for version and variant markers. These bits are fixed, so two hex characters cannot be fully random. The theoretical maximum is about 3.98 bits/char.
