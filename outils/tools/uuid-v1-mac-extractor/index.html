UUID v1 Structure — Bit by Bit

A UUID v1 is 128 bits arranged in a very specific way. Understanding this layout is essential to extract the timestamp and the MAC address.

6ba7b810-9dad-11d1-80b4-00c04fd430c8
├────────────── Timestamp 60 bits ──────────────┤ │v1│ ├─ Clock seq ─┤ ├────── Node 48 bits (MAC address) ──────┤
• 60 bits: timestamp (100‑nanosecond intervals since 15 October 1582)
• 4 bits: version nibble (0001 = 1)
• 14 bits: clock sequence
• 2 bits: variant (10)
• 48 bits: node = MAC address (or random, if multicast bit = 1)

The 60 timestamp bits are not stored sequentially. They are split across three fields — time_low (32 bits), time_mid (16 bits), and time_hi_and_version (12 bits after removing the 4 version bits). To reconstruct the original 60‑bit value, you must reassemble them in the correct order: time_hi (high 12 bits), then time_mid (middle 16 bits), then time_low (low 32 bits). This non‑intuitive layout catches many developers off guard.

Code to Extract Everything — Python, JavaScript, Java

The following functions extract the timestamp, the MAC address, and detect whether the node is a real hardware address or a random one.

Python

import uuid
from datetime import datetime, timezone

def extract_v1_info(uuid_string: str) -> dict:
    u = uuid.UUID(uuid_string)
    if u.version != 1:
        raise ValueError(f"UUID version {u.version} — only v1 contains a timestamp and MAC")
    
    GREGORIAN_OFFSET = 122192928000000000  # 100‑ns intervals between 1582‑10‑15 and Unix epoch
    unix_ts = (u.time - GREGORIAN_OFFSET) / 1e7
    creation_time = datetime.fromtimestamp(unix_ts, tz=timezone.utc)
    
    mac_int = u.node
    mac = ':'.join(f'{(mac_int >> (5-i)*8) & 0xff:02x}' for i in range(6))
    
    # Multicast bit: if the least significant bit of the first octet is 1,
    # the node was randomly generated (not a real MAC)
    is_random_mac = bool((mac_int >> 40) & 1)
    
    return {
        "version": 1,
        "timestamp": creation_time.isoformat(),
        "mac_address": mac,
        "mac_is_random": is_random_mac,
        "clock_sequence": u.clock_seq
    }

# Example
result = extract_v1_info("6ba7b810-9dad-11d1-80b4-00c04fd430c8")
print(result)

JavaScript (browser or Node)

function extractV1Info(uuidString) {
  const parts = uuidString.replace(/-/g, '');
  const version = parseInt(parts[12], 16);
  if (version !== 1) throw new Error(`UUID v${version} — only v1 contains timestamp and MAC`);

  // Reassemble timestamp: time_hi (12 bits), time_mid (16 bits), time_low (32 bits)
  const timeHi  = parseInt(parts.slice(12, 16), 16) & 0x0fff;
  const timeMid  = parseInt(parts.slice(8,  12), 16);
  const timeLow  = parseInt(parts.slice(0,   8), 16);
  const timestamp = BigInt(timeHi) * 281474976710656n +
                    BigInt(timeMid) * 4294967296n +
                    BigInt(timeLow);

  const GREGORIAN_OFFSET = 122192928000000000n;
  const unixMs = Number((timestamp - GREGORIAN_OFFSET) / 10000n);

  // Extract MAC (last 12 hex chars)
  const macHex = parts.slice(20);
  const mac = macHex.match(/../g).join(':');
  const isRandomMac = (parseInt(macHex.slice(0,2), 16) & 0x01) === 1;

  return {
    timestamp: new Date(unixMs).toISOString(),
    mac_address: mac,
    mac_is_random: isRandomMac,
    clock_sequence: parseInt(parts.slice(16, 20), 16) & 0x3fff
  };
}

Java

import java.util.UUID;
import java.time.Instant;

public class V1Extractor {
    private static final long GREGORIAN_OFFSET = 0x01b21dd213814000L; // 122192928000000000 * 100ns

    public static void extract(UUID uuid) {
        if (uuid.version() != 1) {
            throw new IllegalArgumentException("Not a v1 UUID");
        }
        
        // Timestamp in 100‑ns intervals since 15 Oct 1582
        long timestamp = uuid.timestamp();
        long unixMillis = (timestamp - GREGORIAN_OFFSET) / 10_000L;
        Instant instant = Instant.ofEpochMilli(unixMillis);
        
        // Node = MAC address (48 bits)
        long node = uuid.node();
        StringBuilder mac = new StringBuilder();
        for (int i = 5; i >= 0; i--) {
            int octet = (int) ((node >>> (i * 8)) & 0xff);
            if (mac.length() > 0) mac.append(':');
            mac.append(String.format("%02x", octet);
        }
        
        boolean isRandomMac = ((node >>> 40) & 1) == 1;
        
        System.out.println("Timestamp: " + instant);
        System.out.println("MAC: " + mac);
        System.out.println("Random MAC: " + isRandomMac);
    }
}

RGPD and Personal Data — A Real Concern

Under the RGPD (GDPR), Article 4 defines personal data as any information relating to an identified or identifiable natural person. A MAC address is tied to a device, and in many contexts, to a person. The CNIL (French data protection authority) has consistently treated MAC addresses as personal data in multiple decisions, including cases involving Wi‑Fi tracking and device fingerprinting.

If your application generates UUID v1 and exposes them in public APIs, logs, or exports, you may be processing personal data without realizing it. The embedded MAC address can link actions to a specific machine and, in corporate environments, to an individual employee. Consult your DPO and consider migrating to UUID v4 (purely random) or UUID v7 (timestamp + randomness, no MAC).

The Critical Edge Case — When the MAC Is Random

Since RFC 4122, Section 4.5, implementations are allowed to generate a random 48‑bit node instead of using the real MAC address. Python has done this by default since Python 3.7. Java's UUID.randomUUID() generates v4, not v1. The detection is simple: if the least significant bit of the first octet (the multicast bit) is 1, the node was randomly generated and does not correspond to a physical device.

The code functions above already implement this check via mac_is_random. Our embedded tool also displays it clearly in the results — you'll see either "Real MAC address" or "Random node (not a real MAC)".

Frequently Asked Questions

Can I extract the timestamp from a UUID v1 as well?

Yes. The first 60 bits encode the creation time with 100‑nanosecond precision. Both the tool above and our full UUID decoder display it.

Is UUID v1 deprecated?

RFC 9562 (May 2024) recommends UUID v7 over v1 for new applications. v1 is still valid but its privacy characteristics make it unsuitable for public‑facing identifiers.
